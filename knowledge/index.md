# AI人狼 知識ベース

## 概要

ゲームから得られた**事例**を蓄積し、次のゲームの参考情報とする。

**重要**: ここに記録されるのは「教訓」ではなく「事例」。
断定的な一般化は避け、状況依存であることを前提に参照する。

## 運用方針

### 記録するもの
- 何が起きたか（事実）
- 結果
- 考えられる要因（仮説）
- 信頼度（低/中/高）
- 見直しトリガー

### 記録しないもの
- 「〜すべき」という断定的な教訓
- 少数の事例からの過度な一般化

### 削除・修正のルール
- 反例が出たら見直し
- 信頼度「低」は仮説として扱う
- 3ゲーム以上の観察で信頼度を上げる

## 構造

```
knowledge/
├── index.md              # このファイル
├── tactics/              # 戦術考察（ゲーム前）
│   ├── analysis-framework.md  # 思考フレームワーク
│   └── compositions/     # 構成別の考察
│       └── 5p-v2o1w1m1.yaml   # 5人村考察
├── logs/                 # ゲームログ（リザルト）
│   ├── game-001.yaml     # 3人構成テスト
│   ├── game-002.yaml     # 5人構成テスト（人狼潜伏）
│   └── game-003.yaml     # 5人構成テスト（人狼占いCO）
├── joseki/               # 定跡集
│   └── index.md          # 有力な行動とその理由
└── algorithm/            # 思考アルゴリズム
    └── index.md          # 状況判断のアルゴリズム
```

## 4層構造の役割

| 層 | 目的 | 記録するもの |
|----|------|-------------|
| tactics/ | 構造的問題の事前排除 | 思考プロセス、破綻パターン、メタ循環 |
| logs/ | ゲームログ | 実際に起きた事例、想定外の行動 |
| joseki/ | 定跡（パターン） | 有力な行動とその理由、リスク |
| algorithm/ | 思考アルゴリズム | 状況判断、相手の読み、意思決定 |

**運用フロー**:
1. 新しい構成を試す前 → tactics/ で戦術考察
2. ゲーム実行（joseki/ + algorithm/ をプレイヤーに注入）
3. logs/ に事例記録
4. tactics/, joseki/, algorithm/ にフィードバック

## 観察された現象

### 狂人・人狼の連携問題

**観察** (game-002):
- 狂人と人狼はお互いを識別できない
- 人狼が潜伏すると、狂人に手がかりがない
- 狂人がグレーに投票すると、人狼を吊るリスクがある

**仮説**:
- 狂人は占いローラーを優先すべきかもしれない
- 人狼は占い師COした方を支持して連携を作れるかもしれない

**信頼度**: 低（1ゲームのみ）

### 人狼の発言パターン

**観察** (game-002):
- 「占いローラー回避 + グレー誘導」が狼ムーブとして検出された

**仮説**:
- このパターンは人狼の典型かもしれない

**信頼度**: 中（村人が同じ主張をする場合もある）
